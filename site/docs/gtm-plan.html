01
Category & Positioning

What Agent Shield is — and is not.

Positioning is not about what you do. It is about the category you own in the buyer's mind and the alternatives you replace.

Category

AI Agent Governance / AI Risk Infrastructure

The emerging category where security teams need answers before AI agents cause incidents. Agent Shield is the tooling layer that makes this tractable.

Competitive Frame

"The Vanta for AI agents"

Self-serve, transparent pricing, developer-friendly. Compliance tooling that doesn't require a six-month enterprise sales cycle to evaluate.

Primary Tagline

"Know what your AI agents can do before they do it."

Addresses the core anxiety: agents are deployed and nobody knows their actual permissions, blast radius, or guardrail coverage.

Proof Point

Like npm audit, but for autonomous AI.

Every developer knows npm audit. Zero explanation needed. Instant comprehension of the value proposition for the target buyer.

Not positioning as
  • AI firewall
  • AI security scanner
  • copilot
  • general AI governance platform

Sales Motion

Bottom-up, not top-down.

The developer adopts first. The security team buys second. This is the Datadog/Snyk playbook, not the IBM/Palo Alto playbook. No six-month procurement cycles.

Free CLI
$0 / forever
Self-serve

No sales. No signup. Just cargo install. The entire value of the free tier is distribution, not conversion. It seeds the funnel.

Team
$1,500 / mo
Self-serve

Credit card signup. Automated onboarding. No sales call required. Target buyer is the engineering manager or security lead who saw the CLI output internally.

Business
$5,000 / mo
Light touch

Self-serve or 1–2 discovery calls. Target buyer is the CISO or security director. Deal cycle is days to two weeks, not months.

Enterprise
$15,000 / mo
Sales-assisted

2–4 week deal cycle with a defined process. Not 6 months. Target buyer has regulatory obligations (EU AI Act, ISO 42001). Procurement is not the entry point.

Key Insight: The Playbook Matters

Agent Shield's sales motion is bottom-up (developer adopts → security buys), not top-down (CISO evaluates → procurement approves). The difference is measured in months of deal cycle time and tens of thousands in CAC.

06
Content Strategy

ADHD-compatible. Batch-friendly. No content calendar.

Sustained daily marketing is not the plan. Sprint, batch, ship, rest. The GitHub README is the primary marketing asset. Everything else amplifies it.

📝

Batch Blog Posts

Write 3 posts in one focused sprint. Schedule for weekly release. Each post follows the "I scanned X and found Y" format — data-driven, specific, shareable. No opinion pieces.

ℹ️

GitHub README First

The README is the homepage, the sales deck, and the technical documentation simultaneously. It is maintained obsessively. Everything links to it. It never goes stale.

🎤

Conference Talks

One talk per quarter: RSA, BSides, DevSecOps Days. A good conference talk generates inbound for 12 months. Prepare once, benefit for a year.

Explicitly not doing
  • newsletter
  • content calendar
  • daily social media
  • LinkedIn thought leadership posts
  • weekly YouTube videos
  • cold email sequences
